1. FIRST FLASH

Use a Chromium-class browser with Web Serial.

Chrome, Edge, and Brave work well here. Safari and Firefox do not support this path.

2. AFTER BOOT

Use the admin UI for OTA and runtime iteration.

Once the board is up, most upgrades should go through the admin UI or OTA manager rather than repeated wired flashes.

3. MANUAL PATH

Every release still publishes raw artifacts.

Bootloader, partition table, OTA data, app image, bundle zips, and `flasher_args.json` are published for manual bring-up or debugging.
